# 云分享功能业务逻辑总结

云分享 (`CloudShare.tsx`) 是一个集成了文本分享与文件/文件夹分享的综合性功能模块。以下是该模块的详细业务逻辑总结。

## 1. 核心架构与技术栈
- **前端框架**: React (配合 Framer Motion 进行动画过渡)。
- **UI 组件**: Lucide React (图标)。
- **状态管理**: 
  - `activeTab`: 控制当前显示的是“新建分享” (`upload`) 还是“分享管理” (`list`)。
  - `shareType`: 用户正在创建的分享类型（`text` 纯文本 或 `file` 文件）。
  - `textContent` / `selectedFiles`: 分别存储待分享的文本内容或文件列表。
  - `shares`: 存储从后端拉取的所有历史分享记录。
  - `editingShare`: 存储当前正在编辑的分享对象。

## 2. 数据模型
- **ShareContent**:
  - `id`: 分享的唯一标识符（如 `9elb11hd`）。
  - `type`: 分享类型 (`text` 或 `file`)。由于历史兼容原因，可能存在无 `type` 但有 `content` 的旧数据，前端通过 `isText` 逻辑进行兼容。
  - `content`: 文本分享的具体内容。
  - `files`: 文件分享的详细列表 (`FileItem[]`)。
  - `totalSize`: 文件分享的总大小。
  - `name`: 分享的自定义名称。
  - `createdAt` / `updatedAt`: 创建和修改时间。

## 3. 主要业务流程

### 3.1 导航与视图切换
- 页面顶部提供了 `+` 图标按钮和“分享管理”按钮。
- 点击 `+` 切换到上传视图 (`upload`)。
- 点击“分享管理”切换到列表视图 (`list`)，并触发数据拉取 (`fetchShares`)。

### 3.2 创建分享 (新建区)
**界面设计**：文本输入框与文件上传框同屏展示，不再使用 Tab 分离。
- **文本分享**:
  - 用户在 `textarea` 中输入内容。
  - 自动触发 `setShareType('text')`。
- **文件分享**:
  - 提供拖拽区域和点击上传功能。
  - 利用原生 `<input type="file" multiple />` 支持多文件上传。
  - 利用原生 `<input type="file" webkitdirectory directory />` 支持整个文件夹上传。
  - 选中文件后，自动触发 `setShareType('file')`。
  - 显示已选文件数量、总大小，并提供“清空重选”按钮。
- **提交逻辑 (`createShare`)**:
  - 根据当前的 `shareType` 执行不同的 API 请求。
  - **文本**: 发送 JSON 数据 `{ content, name }` 到 `/api/share/text`。
  - **文件**: 使用 `FormData` 封装文件流和名称，发送至 `/api/share/files`。
  - 成功后清空表单并切换回管理列表。

### 3.3 管理与展示 (列表区)
**数据展示**：采用数据表格布局（Grid Layout）。
- **数据加载**: 通过 `/api/share/list` 接口拉取历史数据。
- **列表项展示**:
  - **ID 列**: 等宽字体标识。
  - **类型列**: “文本”标签或“文件”标签（兼容旧数据）。
  - **内容预览**: 
    - 文本类型截断显示正文。
    - 文件类型显示文件/文件夹图标、名称、文件数量和格式化后的总大小。
  - **创建时间列**: 本地化的时间字符串。
  - **操作列**:
    - **复制**: 拼接生成 `https://domain/s/{id}` 格式的短链接并写入剪贴板。
    - **编辑**: 仅对文本类型可用。点击打开编辑弹窗。
    - **删除**: 弹出浏览器原生 `confirm` 确认后，调用 `/api/share/{id}` (DELETE) 接口。

### 3.4 文本编辑 (弹窗)
- 通过 `AnimatePresence` 实现弹窗动画。
- 允许用户修改现有的文本分享名称和正文内容。
- 点击“保存更改”后，调用 `/api/share/{id}` (PUT) 接口更新数据，成功后自动刷新列表。

## 4. 后端路由联动 (依据全栈架构)
- 前端列表展示的预览链接格式为 `/s/:id`。
- 后端在收到 `/s/:id` 请求时：
  - 如果是文本，直接拦截并返回 `text/plain` 格式的内容。
  - 如果是文件，发送 `302 Found` 重定向至前端 `/share-preview/:id` 路由，由 SPA 渲染文件树及提供打包下载功能。
